# 多模型研究/研發流水線 · 複用 Playbook（v1.2 ｜ 論文 + 量化策略 兩剖面）

> 從《中國上市公司併購重組成功案例研究》8 階段實踐 + 開卷/盲答對照實驗中提煉、並經姊妹項目《形態匹配ETF策略》18 版交叉驗證的可移植方法。
> 目標：把"多個獨立模型交叉協作**生產 + 獨立評審**"這套方法，搬到你下一個真實項目上——無論是**一篇論文**，還是**一條量化策略**。
> 本版**通用、任務自適配、不綁定具體選題和模型**：**共享一個方法核心**（五條鐵律 + Phase 0 選題/否決 + 溯源門禁 + 評審輪換 + 診斷→修復閉環），下掛**兩個執行剖面**——論文剖面（§3 階段總覽）與量化策略 R&D 剖面（§11）。
> 注：三件文件名仍叫 `多模型论文流水线_*` 是歷史原因（保持引用穩定）；內容自 v1.2 起已泛化爲研究/研發通用。

## 0. 怎麼用這份包

本複用包三件：
- **`多模型论文流水线_playbook.md`**（本文）— 方法手冊：原理、鐵律、階段總覽、按類型特化的承重菜單。
- **`多模型论文流水线_playbook.json`**— 同內容的機讀版，便於下個項目程式化讀取階段/門禁。
- **`阶段模板件.md`**— 每個階段一套參數化 prompt+config 骨架，`{{占位符}}`填空即用。

**上手三步**：①讀本文第 2、4 節（鐵律 + 角色分配原則）；②按第 5 節確定你論文類型的承重階段、裁剪掉用不上的；③去 `阶段模板件.md` 複製對應骨架、把 `{{}}` 填成你的選題，按命名規範存成 `phase{N}_{model}_{prompt.md,config.json}`。

## 1. 核心信念（爲什麼值得這麼折騰）

一句話：**一個模型對自己的工作做自評有系統性盲點；多個獨立模型反向交叉審查，能抓出自評看不見的硬傷。**

本項目的實證：總裝者一度自評執行度 106%，而獨立盲審判"退回重寫(68)"，抓出了學術誠信紅線（模擬數據標成真實來源）這種自評完全沒暴露的問題。開卷/盲答對照進一步量化出：多模型答辯產出的"深度"大體是真的（認知維度盲答下零衰減），不是模型互相吹捧的表演。

**這套方法貴在"獨立性"，不在"模型更聰明"。** 所有設計都服務於一件事：製造真正獨立的視角，並防止任何一個視角既當運動員又當裁判。

## 2. 五條鐵律（跨論文類型都不變）

1. **每階段配 `prompt.md` + `config.json` 雙件。** 人機皆可讀，防提示詞漂移、可追溯、可重跑。
2. **沒有任何模型審查/評分自己做過的環節。** 自審有盲點，且會爲舊決策辯護。撰稿人不審自己的稿、出題人不評自己的題。
3. **出題人與評分人分離；評分人選"零捲入"模型。** 出題人批自己出的題永遠有偏差；評分官要結構性獨立（沒參與過任何前序環節）。
4. **審覈早於撰寫，總裝最後做。** 領域硬傷要在動筆前攔截；總裝者必須擁有全局上下文才能統一處理。
5. **誠信紅線 + 內部一致性是不可談判的。** 模擬/估算數據絕不標成真實來源；每個數字登記來源等級；報告自身也要過一致性自檢（連成績單都犯過"判語分數≠計算分數"的 bug）。

## 3. 階段總覽（Phase 0–9）｜ 論文剖面

> ⚑ 標記的 **Phase 0** 和 **Phase 9** 是本項目復盤後**新增**的兩段——上一輪的教訓是“病根在設計階段沒立規則”和“診斷出一堆 bug 卻沒改回論文”。這一版把它們補上。**v1.1 又給 Phase 0 補了第二半「事前否決位」**：來自姊妹項目（形態匹配ETF，一個量化策略 R&D）的教訓——一套擅長抓執行錯誤的評審機制，救不了一個本該第一天就被否決的前提（詳見 §6.5）。

| Phase | 角色原型 | 它幹什麼 | 關鍵門禁/產出 |
|------|---------|---------|-------------|
| **0 ⚑** | 選題與規則鎖定 **+ 事前否決** | ①定選題、研究問題、成功/操作化定義；②**前置鎖定數據溯源規則 + 文獻時效規則**；③**由獨立角色做「理論 kill-test」**——核心假設寫成可證偽命題、給出最強反方論證、跑「最便宜的證偽實驗」、預登記「死亡判據」 | 選題書 + 溯源四級規則 + 文獻硬約束 + **否決官報告（最便宜證偽實驗結果 + 死亡判據 + GO/STOP）**，均寫成可校驗條款 |
| **1** | 方案設計師 | 出大綱、候選案例/數據、理論框架、文獻清單 | 設計藍圖；理論須能逐案例/逐假設對位，不是擺設 |
| **2** | 領域專家審覈 | 在撰寫前核領域硬傷（準則/法條/識別策略/數據口徑） | 領域審查報告；價值最高的一環，抓 Phase 1 的硬傷 |
| **3** | 撰稿人 | 寫正文主力，**留佔位符**給下游 | 初稿；用 `【数据待填】/【图表N】/【编者注】` 不硬塞數據 |
| **4** | 總裝/交付 | 數據補全、排版、終審、佔位符統一處理 | 總裝稿；總裝者擁有全部上下文 |
| **5A/5B** | 交叉盲審 ×2 | 兩個獨立模型**反向**審自己沒做過的環節 | 兩份獨立盲審（互不溝通）；按研究生標準 |
| **6** | 整合裁決與修訂 | 收雙盲審→意見映射→衝突裁決→統一修改→修訂稿→終審 | 修訂版 + 裁決記錄；終審三檔 |
| **7 ⚑** | 設計回溯 | 以原設計者身份評執行度，把殘餘問題的病根**上移到設計階段** | 復盤報告；最有價值的產出是"該在 Phase 0/1 立什麼規則" |
| **8** | 答辯模擬 | 出題(×3分層)→學生作答→零捲入評分 | 題單 + 答辯記錄 + 成績單 |
| **9 ⚑** | 答辯後定稿 | 把 5/6/8 診斷出的具體缺陷**真正改回論文**，或封存時列明"已知缺陷+不修原因" | 終稿 vX，或誠實的 closure note |

**裁剪原則**：這是全套骨架，不是每篇論文都要全跑。文獻綜述類可弱化 Phase 8 的方法論出題；時間緊時 Phase 7 可併入 Phase 6。但 **Phase 0、2、5、9 不建議砍**——它們分別守著“規則前置 + 事前否決”“撰前攔硬傷”“獨立盲審”“閉環修復”，是這套方法的命門。

## 4. 角色分配原則（任務自適配，**不綁定模型**）

上一個項目用 Kimi/GLM/GPT/Claude/Qwen 各擔一角，但那是**針對那個任務**的最優解，不是固定公式。換論文、換手頭模型時，按下面三條重配：

1. **把每個模型放到它已被你驗證最強的環節。** 例：擅長長鏈推理/整合的做總裝+裁決；領域知識最紮實的做領域審；文筆與鋪陳最好的做撰稿；最瞭解全局結構的做答辯主席。
2. **強制輪換以滿足鐵律 2/3。** 誰撰了稿，就不能審這稿；誰出了題，就不能評這題。讓模型去審"自己沒做過的環節"（上輪讓撰稿模型去審領域、讓審領域的模型去審寫作，正是爲此）。
3. **評分官留給"全程零捲入"的那個模型。** 它沒出過任何題、沒寫過任何稿，才有外審專家式的結構性獨立。

> 記法：角色是"槽位"，模型是"填進槽位的人"。Playbook 定義槽位與槽位間的隔離規則；每個新項目重新分配人選。

## 5. 按論文類型特化"承重階段"（你選了通用版，這裏是三套菜單）

選題定了之後，按類型把對應門禁掛到 Phase 0/2/8：

### A. 實證/量化研究（最貼你的量化背景）
承重 = **數據與識別**。重點門禁：
- 數據溯源四級 + 機器校驗（見第 6 節），**接真實數據前不得下實證顯著性結論**。
- 識別策略：有沒有反事實/對照組？單樣本前後對比 ≠ 因果，措辭用"伴隨"非"帶來"，必要時上 DID/合成控制並交代匹配難點。
- 事件研究陷阱（若用）：t 值構造（CAR/(σ_AR·√L)，單一估計窗一個 σ）；A股停牌→事件窗日曆錯配、復牌連續漲停→AR 正自相關致 t 虛高；單因子 vs FF3/Carhart 在風格行情下會把因子收益誤記爲效應；測收購方還是標的方要分清、基準指數要匹配被測市場。
- 指標體系含**現金流維度**（利潤可粉飾、經營現金流難造假）；跨主體比較先做行業基準化，別拿一把尺子量不同行業。
- 穩健性的實質是**改方法**（市場模型→多因子、CAR→BHAR、參數→非參/bootstrap），**不是改窗寬**；"窗口越長效應越大"是洩漏警訊不是穩健證據。
- 杜邦等會計恆等式只能**描述**不能**證因果**；口徑（歸母 vs 全部權益等）全文統一，否則因果歸因失分。

### B. 案例研究
承重 = **選擇邏輯與比較**。重點門禁：
- 案例結構對稱性：成功案例要不要失敗/中止案例做對照？"成功"的操作化定義在 Phase 0 寫死，別事後合理化。
- 區分會計/口徑效應 vs 真實經濟效應（如並表增長 vs 協同增長）。
- 跨案例結論強度：典型成功 ≠ 可推廣；措辭從"證明成功"降到"成功的必要條件"。

### C. 文獻綜述/理論
承重 = **文獻與論證結構**。重點門禁：
- 文獻時效門禁（如近5年 ≥30%、核心期刊佔比），Phase 0 立規則、Phase 5 校驗。
- 檢索策略可復現（庫、關鍵詞、納入/排除標準、檢索日期）。
- 論證是"梳理"還是"建構"——理論框架要能產出可檢驗命題，不是文獻堆疊。

## 6. 數據溯源門禁（貫穿全程，**Phase 0 前置**）

上一項目最大的傷口、被三個獨立信源（盲審/復盤/盲答對照）反覆指向的同一處。規則：

- **四級來源標識**：`[R]真实`（數據庫/年報可複核）/ `[S]模拟`/ `[E]估算`/ `[P]公开事实`。每個數字登記等級。
- **發布門禁**：`[S]/[E]` 不得支撐實證顯著性結論；只能做"方法演示"。摘要/正文/圖表口徑必須一致（防"摘要289.34 vs 表格289.04"這類內部矛盾）。
- **可復現鏈**：原始 CSV（不可改）→ 清洗腳本（可重跑）→ 表格單元格，每個 `[R]` 能反查到源文件某行某列。
- **機器校驗**：寫一個能跑的校驗器，掃到"`[S]`標成`[R]`"或"摘要/正文數字打架"就報錯，擋在發布前。
- **前置**：以上不是 Phase 6 的事後補丁，是 **Phase 0 的設計件**。這是 Phase 7 復盤最硬的結論——誠信紅線和文獻時效的病根都在“設計階段沒立規則”。

## 6.5 事前否決位：選題/假設的理論 kill-test（Phase 0 前置）

> v1.1 新增的 Phase 0 第二半，來自**姊妹項目「形態匹配ETF策略」**的教訓。Phase 0 原本只管“立規則”（溯源/時效），但兩個項目合起來暴露了一個更靠前的漏洞：**最貴的錯誤不在任何一步執行裏，而在“根本該不該做”這個沒人在事前問的問題上。**

**爲什麼需要它（兩項目的同一根病）**
- 併購重組：以“研究論文”立項，但起手框架根本不是研究框架（是教科書綜述）。沒人在動筆前問“這到底是不是一個研究問題”。
- 形態匹配ETF：核心假設“形態匹配能預測收益”在理論上註定失敗（DTW 的時間彎折容忍恰好抹掉了預測最需要的方向性資訊——一個白板 30 分鐘能講完的反方論證），實證上從 V2 就負 ICIR，卻**燒了 18 個版本**才在終期點破，“放棄判據”也拖到第 14 個文件夾才寫下。
- 共性：流水線在**執行質檢**（抓 bug、抓過度外推、閉環修復）上已被反覆證明有效，但它**結構性地看不見“premise 該不該做”**——所有評審都在問“這一步做對了嗎”，沒人被指派去問“這件事根本該不該做”。

**它是什麼：一個獨立的「否決官」角色，在燒工程/動筆前先試著把選題殺死。** 四個動作：
1. **把核心假設寫成可證偽命題**：不是“研究 X”，而是“X 滿足可被數據推翻的具體斷言 Y”。
2. **給出最強反方論證**：站在“這注定不成”的立場，寫出最鋒利的理論 kill（ETF 那條 DTW 論證就是範例）。能 30 分鐘講完的“此路不通”，絕不留到第 18 版。
3. **設計並跑「最便宜的證偽實驗」**：用最小成本（一段腳本、一張圖、一個邊際相關性）先打它一槍，**先於**全面建設。
4. **預登記「死亡判據」**：把“出現什麼結果就放棄”寫成可判定條款，**在拿到結果之前**寫死——否則事後一定會被合理化成“再迭代一版試試”。

**門禁（不可談判）**
- 否決官**必須是獨立角色**——優選全程零捲入的那個模型（鐵律 3 的同一個人），或專設紅隊；**絕不能是選題設計者本人**（他對這個想法有承諾，殺不下手）。
- **最便宜的證偽實驗若已失敗 → 停，不進入 Phase 1，不開工。** 把“誠實的負結果”在第 0 階段就拿到，遠比燒十幾版後再拿到便宜。
- 若透過 → 繼續，但**死亡判據全程掛著**，每輪迴測/每稿對照它一次；觸線即停，不許移動球門。

**和其它階段的關係**
- 它把 Phase 7「病根上移設計階段」的**事後**動作，變成 Phase 0 的**事前**動作——Phase 7 問“早該立什麼規則”，Phase 0 否決位問“早該問這能不能成”。
- 它和“改一個測一個”互補：後者優化**區域性執行**，但對一個註定的前提無能爲力；否決位負責**全局前提**。兩者缺一不可。

## 7. 答辯模擬 + 開卷/盲答對照協議

**答辯出題分三層、三個模型**：主席（設計層，應是最懂論文骨架的那個模型，不是最聰明的）、領域專家（準則/方法層）、方法論專家（實證/統計層）。三人獨立出題、領域不重疊。學生（撰稿模型，對論文有作者級熟悉度但沒參與審稿）逐題答。評分官（零捲入模型）雙層評分（逐評審維度60% + 跨維度綜合40%）+ 自檢錨定。

**開卷/盲答對照（評價器的自校準，強烈建議保留）**：
- 學生作答時**必須對考官的「考察意圖/預期回答方向」盲**——只給純題幹。否則是開卷應考，分數虛高、測的是"複述評分細則"而非真實理解。
- 跑兩遍：開卷（含答案鍵）+ 盲答（剝答案鍵），**分差 = 答案洩漏污染量**。本項目量出僅 ≈3 分且全在"操作性回憶"層、認知維度零衰減——這才證明了多模型答辯的深度大體真實。
- 協議要點：評分官兩階段——Stage 1 不看對照基線先冷評鎖分，Stage 2 才算分差（防錨定）。

## 8. 內部一致性 & 誠信紅線（self-check，別隻盯論文）

- **報告自身也要過一致性自檢**：本項目兩份成績單都犯過"判語寫的分 ≠ 最終表算的分""加權算式記法錯"的 bug。凡是有數字結論的產出，發布前自己核一遍勾稽。
- **誠信紅線**：模擬數據絕不冒充真實來源（這是上一項目被判"學術不端嫌疑"的根因）。寧可論文更短、結論更弱，也要每個數字誠實標級。
- **遞迴懷疑**：連評分官自己都有盲點（它給方法論判"零紅利"，但記錄顯示盲答確實丟了精度）。所以"誰來監督監督者"沒有終點——保留一層人工/獨立複核。

## 9. 診斷→修復必須閉環（上一項目栽在這）

## 11. 第二執行剖面：量化策略 R&D（論文之外的另一條產線）

> v1.2 新增，來源：姊妹項目《形態匹配ETF策略》18 個版本的實踐。**共享核心不變**（五條鐵律、Phase 0 選題+否決、溯源門禁、評審輪換、診斷→修復閉環），變的只是**執行半段的階段**和**領域門禁**——論文的"撰稿/盲審/答辯"換成策略的"編碼/回測/消融/冪等"。

### 11.1 爲什麼是"剖面"而不是"另一個包"（決策記錄）

> 這一節本身是 v1.2 的"爲什麼這麼做"記錄——免得日後有人（包括你自己）再糾結要不要 fork 一個獨立的策略包。

- **採用（A）**：把策略 R&D 作爲現有包的**第二執行剖面**寫進來，只固化兩樣最值錢、最易流失的東西——**階段骨架 + 量化專屬失敗模式清單**；**不**預製整套 prompt/config 模板。
- **不選（B）完整重構 + 寫全策略模板**：那會造出一**批從沒在真任務上跑過的模板**——正是 ETF 項目用 18 個版本警告的"驗證前提之前就燒工程"。模板該在你真起一個策略項目時、對著真任務實例化，不是現在憑空寫。
- **不選（C）什麼都不做、等真項目再說**：策略 R&D 的**失敗模式清單**是用 18 個版本踩坑換來的、**會過期**的知識——現在不寫下來，下次就得重踩一遍。它便宜、召回價值高，所以"什麼都不做"會白白丟掉它。
- 一句話：**固化會過期的（清單+骨架），推遲沒驗證的（模板）。** 這本身就是 §6.5 否決位精神用在"要不要建這個包"上的一次應用。

### 11.3 量化策略專屬門禁清單（18 個版本買來的，最易流失，務必帶走）

> 這是 §5 菜單 A（實證/量化）向**可執行策略 R&D（程式碼 + 回測）**的延伸；重疊處，§5 是論文框架、本節是程式碼/回測框架。

**（1）洩漏 / 未來函數（最致命、最隱蔽）**
- **標籤-執行對齊**：標籤用 T 日資訊，執行就必須 T+1 可得（ETF 栽過"標籤用 T 收盤、實盤 T+1 開盤"的系統性錯配）。
- **滾動窗含當日**：`x[-252:]` 若含當日就是未來函數，應 `x[-253:-1]`。
- **in-sample 回退**：NaN/異常分支偷偷回退到全訓練集 in-sample 預測；Stacking 元學習器在訓練集自身 `predict_proba` 生成元特徵 = 洩漏。
- Walk-Forward 用**固定窗**不用擴展窗；每折**獨立** fit scaler。

**（2）冪等性 / 程式碼紀律**
- 信號層必須**確定性**（IC/ICIR/特徵重要性重跑一致到小數點後 N 位）；執行層非確定性（撮合引擎）要**量化出噪聲底**，而非假裝沒有。
- **改一個常量，全局搜聯動點**：ETF 把 `IC_WINDOW` 60→12 卻漏改就緒閾值 `>=20` → 整個 IC 體系變死程式碼。
- `except: pass` 是 **P0 級程式碼異味**，能讓致命 bug 跨多版本隱身。
- **單位一致**：`VOL_ABSOLUTE_CAP`（日度 σ）拿去和年化波動率比 → 100% 或 0% 觸發。

**（3）實驗設計**
- **改一個測一個**：單次只動一個模組，否則無法歸因（ETF V3.2 同改三模組 → 全版本最差且無法定位）。
- **消融矩陣**：全特徵 vs 各子特徵組，找出誰真有貢獻、誰是噪音；當心把"線性加性稀釋"誤讀成"特徵拮抗"。
- 消融/審計報告**自身**也要被獨立複核（ETF R5 初版填錯數據、謊報程式碼邏輯，複核後結論反轉）。

**（4）信號有效性 / 遷移檢驗（最高價值的一條）**
- **遷移檢驗**：把信號搬到**另一個 universe** 驗證（ETF→個股）。ETF 上 sim_variance RankIC +0.1363，個股上 −0.0104、整體 ICIR +0.0002——**指數上的"信號"很可能是成分股聚合的統計假象**。這是抓"假 alpha"最便宜最狠的一刀，強烈建議每個信號都過一遍。
- **顯著性**：點估計不可信，要 t 檢驗/置信區間；"無法拒絕 IC=0" ≠ "證明 IC=0"，別反向過度宣稱。
- 多版本篩選放大數據挖掘偏差 → Deflated Sharpe / White Reality Check / 至少宣告"這是第 N 次試驗取最大值"。

**（5）穩健性 / 歸因 / 成本**
- **穩健性 = 改方法**（市場模型→FF3、CAR→BHAR、參數→bootstrap），**不是改窗寬**；"窗口越長效應越大"是洩漏/漂移警訊。
- **歸因要拆**：把 Beta + 風控擇時 從"信號 alpha"裏剝出來（ETF 的 4.55% 主要是 Beta+風控，真正的形態 alpha <1pp）；別把通用風控層的功勞算到你的賣點上。
- **成本現實**：換手率 × 手續費要進賬（ETF 10 倍換手、手續費吃掉收益 8.5%）；指標含**現金流維度**（利潤可粉飾、現金流難造假）。
- **基線**：永遠對比被動基準 + 一個樸素替代（如"貨基+指數"）；低迴撤可能只是個通用波動率 overlay 的功勞，不是你的策略。
