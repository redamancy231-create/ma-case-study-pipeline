# 数据溯源方案模板（Data-Provenance Spec）

> **适用**：多阶段多模型学术报告流水线（Phase 1 设计 → Phase 3/4 数据落地 → Phase 5 审查 → Phase 6 裁决）
> **目的**：杜绝"构造/模拟数据被标注为真实来源"的学术诚信风险；保证每个数字可溯、可核、可复算。
> **责任**：数据落地阶段（Phase 4）填登记表 → 审查阶段（5-A）核验 → 发布前签字门禁（Phase 6）。

## 1. 溯源等级（每个数据序列必须归入且仅归入一类）

| 代码 | 含义 | 强制标注文本（图表"资料来源"必须含此口径） |
|---|---|---|
| **R 真实** | 取自数据库/年报原始值 | "数据来源：<库名/年报> <版本·年度>，取数日期 YYYY-MM-DD" |
| **S 模拟** | 按合理区间构造，非抓取 | "基于公开信息与行业合理区间构造的模拟数据，仅用于方法演示（非真实数据库抓取）" |
| **E 估算** | 由 R/公开值推导、折算 | "由 <来源> 经 <方法> 测算，非原始披露值" |
| **P 公开事实** | 公告/法规/年报定性事实 | "据 <公告或法规名称及条号>" |

> **红线**：S 类**不得**出现 Wind/CSMAR/年报等真实来源字样，也不得简写为"经作者整理计算"以掩盖其模拟性质；R 类**必须**保留可复算底稿。

| 数据ID | 字段/指标 | 出现位置(表/图/正文) | 等级 | 来源 或 构造方法 | 取数/构造日期 | 底稿链接 | 责任阶段 |

## 3. 标注一致性规则

- 同一序列在**正文/表/图/摘要**四处的等级与来源标注必须一致（一处改、全处改）。
- 图表"资料来源/数据来源"行文字 = 该序列等级的强制标注文本（见 §1）。
- 实证性措辞（"显著""研究发现""x% 水平显著"）**仅 R 类可用**；S/E 类一律改为"演示性/示例值"表述。

## 4. 勾稽校验（发布前自检）

- [ ] 跨表/图/正文数值一致（如商誉余额、资产负债率、净利润）
- [ ] 内部恒等式成立（ROE = 净利率 × 周转率 × 权益乘数；CAR 与事件窗口口径自洽）
- [ ] 单位、口径、基准年（T）全文统一
- [ ] R 类有底稿可复算；S 类有构造规则可复现

## 5. 发布门禁（5-A 审查 + Phase 6 签字，缺一不发）

- [ ] 全部数据序列已登记，无"未登记即入稿"
- [ ] **无 S 类被标为 R 类**（诚信红线）
- [ ] R 类底稿齐备；S 类已在研究局限处显式披露
- [ ] 审查官（5-A）与裁决官（Phase 6）签字 + 日期

## 6. 机器可校验存根（可选，与 docx 并存）

```json
{ "data_registry": [
 {"id": "D-001", "fields": ["roe_crrc_2013_2017"], "locations": ["表4a", "图1"],
 "tier": "S", "source": "行业合理区间构造", "verified_date": "2026-05-28",
 "worksheet": null, "owner_phase": 4,
 "source_note": "基于公开信息与行业合理区间构造的模拟数据，仅用于方法演示"}
]}
```

> **用法**：Phase 4 输出此 JSON 与 docx 并存；Phase 5 校验脚本比对 docx 内"资料来源"文字与 `tier` 是否匹配，自动拦截 S↔R 错配——把本项目这次"模拟标真实"的红线问题在发布前挡住。
