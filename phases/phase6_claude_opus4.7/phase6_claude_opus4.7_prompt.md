### 阶段 A：双审整合与冲突裁决

#### A1. 意见映射

将两份审查报告中的所有修改建议按"章节—段落—问题类型"维度归一化到统一表格：

```
| ID | 章节定位 | 问题类型 | 5-A意见 | 5-B意见 | 关系 | 裁决 |
|----|---------|---------|---------|---------|------|------|
| M01 | 3.1节第2段 | 法律条款引用 | 终稿引用"第173条"，与Phase2核验一致 | （5-B未涉及）| 单方 | 采纳5-A→无需改 |
| M02 | 3.1节第3段 | 引文格式 | （5-A未涉及）| 缺少(作者,年)标注 | 单方 | 采纳5-B→补标注 |
| M03 | 4.2节第4段 | 商誉金额 | 5-A：数值与公开事实不符，建议改为约220亿 | 5-B：数字写法不规范，建议统一为"亿元" | 同点不同维 | 合并：改数值+统一单位 |
| M04 | 5.2节表3 | 数据合理性 | 5-A：ROE与净利润不勾稽 | 5-B：表序与正文引用不符 | 同表不同点 | 各自修订 |
| M05 | 6.1节末段 | 结论支撑 | 5-A：数据不支持"显著改善"表述 | 5-B：结论推导跳跃 | 同向相互印证 | 必改：弱化措辞+补论据 |
| M06 | 1.2节 | 文献引用 | 5-A：Andrade等2001文献引用准确 | 5-B：APA格式"&"位置错误 | 反向 | 采纳5-B（格式越界但属其专业）|
```

#### A2. 裁决原则

| 情形 | 裁决规则 |
|---|---|
| **领域内单方** | 该领域专家提出、对方未涉及 → 默认采纳（5-A 法律/会计/案例/数据；5-B 结构/写作/论证/文献/术语图表）|
| **跨界单方** | 5-A 谈写作、5-B 谈法律 → **降权处理**：若与该领域专家未明确否定，可参考；若与该领域专家明示意见冲突，**不采纳跨界意见** |
| **同向双方** | 两人都指出同一问题（无论同点同维还是同点不同维）→ **必改**，且按"问题更尖锐的一方"措辞执行 |
| **同点反向** | 两人在同一点上意见相反 → 由你独立判断，依据下列优先级：①原文/Phase 2 → ② 评审通用规范 → ③学术常识；裁决理由必须写入整合报告 |
| **互补合并** | 两人从不同角度提同一处问题 → 合并为单一修改项，融合两人意见 |
| **越界否决** | 跨界批评若侵入对方专长（如 5-A 评判文献是否"够档次"、5-B 评判法律条款是否"过时"）→ 标注"越界，不采纳"，并写明理由 |

#### A3. 修改清单生成

按裁决结果形成统一的**修改清单 v1**，每条包含：

```
ID: M-XXX
位置: 章节.节.段（必要时指明句号）
分级: 必改 / 建议改 / 观察
来源: 5-A / 5-B / 双方 / 整合官独立提出
问题描述: ……（具体到字面）
改前: "……"（摘录终稿原文）
改后: "……"（修订建议文本）
裁决依据: ……（一句话说明为何按此修订）
```

**分级口径**：
- **必改**：①双方同向重大问题；②任一方明确"严重偏差/学术不端嫌疑"；③数值/主体/时间事实错误；④引文严重缺失
- **建议改**：①单方重大问题；②双方同向一般问题；③措辞优化
- **观察**：①争议性学术观点；②风格偏好；③可改可不改

### 阶段 B：修订实施

#### B1. 修订两类产物

| 产物 | 处理方式 |
|---|---|

| `generate_docx.py` | **必须同步更新**：数据修订改 DATA 字典；章节文字修订改对应章节生成函数；图表序号变化改图表生成逻辑 |
| 修订后 docx | 通过更新后的脚本重新生成，文件名采用 `中国上市公司并购重组成功案例研究_v2.docx`（保留 v1 终稿作为历史版本）|

#### B2. 修订执行规则

1. **逐条对应**：每实施一条修改清单项，在 changelog 中记录"M-XXX 已执行 / 未执行 / 部分执行"
2. **不擅自扩张**：仅修改清单中明确列出的内容；不顺手"优化"未被批评的段落（避免引入新偏差）
3. **数据修订连锁反应**：若某指标数值被修订，必须沿文本/表格/图表/勾稽链全文回扫，确保前后一致
4. **图表更新策略**：若数据修订涉及图表，必须重新生成 PNG（更新 `figure*_*.py` 或在 `generate_docx.py` 中重绘）
5. **参考文献修订**：补充文献时优先从 Phase 1 候选清单和 Phase 2 补充清单中选取；新增文献需在正文找到至少一处真实引用位置

#### B3. 二次自检

修订完成后，**用 5-A 与 5-B 的检查表自查一遍**：
- 5-A 的法律/会计/案例/数据四维度检查项是否全部落实
- 5-B 的结构/写作/论证/文献/术语图表五维度检查项是否全部落实
- 未落实项必须在终审报告中说明理由（不可静默跳过）

### 阶段 C：终审签字

#### C1. 整合评分

将 5-A 与 5-B 的评分按**加权法**整合：

```
合规与事实层（5-A 出具）权重：60%
学术规范与写作层（5-B 出具）权重：40%

整合得分 = 5-A 得分 × 0.60 + 5-B 得分 × 0.40
```

权重设置依据：会计学本科毕业论文中"事实/合规/数据"是**硬指标**（一票否决可能性高），"写作规范"是**软指标**（可通过修订快速改善）；但 5-B 的权重不低于 40%，因为盲审实践中写作规范缺陷（如缺文献综述、参考文献严重不足、引文格式混乱）足以单独触发"修改后通过"。

#### C3. 终审结论

## 特别说明

### 关于你自己 Phase 4 的工作

- **接受批评**：Phase 4 总装阶段可能因模拟数据、图表布局、引文格式留下硬伤，5-A/5-B 的批评首先针对你的产出。**不要为 Phase 4 的决策辩护**，按裁决规则照单全收。
- **承认局限**：若 5-A 指出模拟数据合理性问题（如 ROE 与净利润不勾稽），优先承认并修订；不要以"这是模拟数据"为由拒改。
- **学术不端红线**：若 5-A 指出"宣称 Wind 来源但实为模拟"构成学术不端嫌疑，**必须**在终审报告中将该问题列为"必改"，并在修订版中明确数据性质（如改为"模拟数据，基于行业合理区间构造"）。

### 关于裁决的纪律

- **不取众数**：5-A 和 5-B 的意见不是投票，是各自专长内的判断。不要用"两位审稿人都没强烈反对"作为不改的依据。
- **不和稀泥**：反向意见不要找"中间道路"应付，必须给出独立裁决理由。
- **不当裁判员又当运动员**：你既是裁决官又是修订执行者，但在裁决阶段必须以"陌生整合官"姿态读两份审查报告，不要为修订便利偏向某一方。

### 关于交付完整性

- 任何"未落实"的修改清单项必须在 changelog 中写明理由（如：5-A 提出但属越界、5-B 提出但终稿原表述已正确等）
- 终审若给"通过"档，必须确保 v2 不存在任何"必改"残余项
- 终审若给"修改后通过"档，必须列出残余必改项清单，让作者后续修订
- 终审若给"退回重写"档，必须说明系统性问题与重写范围
