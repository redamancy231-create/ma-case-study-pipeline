# Phase 8-A：答辩委员会主席质询问题清单

## 角色身份

你在这篇论文的早期阶段设计了研究蓝图（Phase 1），在终稿完成后回溯了设计执行度（Phase 7）。你以答辩主席身份出场——你了解这篇论文的骨骼结构，你的问题是设计层面的，追问"为什么这样设计"和"回头看这样设计对不对"。

---

## 质询问题

### Q1 [研究设计的完整性与案例选择逻辑]

**问题**：Phase 1 的原始蓝图推荐了四个候选案例——中国南车-北车、美的-小天鹅、宝钢-武钢，外加顺丰借壳作为备选——设计意图是构建一个对称的比较骨架：两个同一控制吸收合并加上两个非同一控制下的市场化并购。终稿变成了 2+1 结构（两个同一控制加一个非同一控制跨境并购），而且两个同一控制案例都受国资委主导、政策驱动力高度同质。**你现在回头看，这个 2+1 的结构是会计学比较分析视角下的合适取舍，还是一种结构性的遗憾？** 如果必须加一个第四案例来修复结构，你会选什么类型——境内非同一控制收购、借壳上市，还是失败案例作为对照组？你的选择逻辑是什么？

**与论文设计的关系**：普通评审只能问"你为什么选这三个案例"，但我知道 Phase 1 原本设计了四案例对称结构，也清楚案例二因会计分类错误被替换为库卡后，论文在"非同一控制"维度上只剩一根独苗，且两个同一控制案例的政策同源性问题在 Phase 1 中并未被预先识别。

**考察意图**：测试学生对案例结构平衡性的自觉意识，以及其是否理解"案例选择不是凑数，而是为比较分析服务"的方法论原则。

**预期回答方向**：好的回答应主动指出"两个同一控制案例政策驱动力重叠"削弱了因果推断的锐度；能解释库卡替换小天鹅后虽然保住了非同一控制维度，却失去了境内市场化并购的对照；若提议第四案例，需给出与现有三案例形成因果互补而非简单类型填补的逻辑。仅回答"时间有限所以三个够了"或"库卡案例更有意思"属于不合格。

### Q2 [学术贡献、理论框架与文献对话]

**问题**：Phase 1 在 2.2 为你设计了四个理论支点——效率理论、代理理论、市场势力理论和交易费用理论——并要求终稿在分析中予以回扣。终稿在 6.1 确实回扣了，但回扣方式是概述性的：一段话把四个理论各提了一句，没有逐案例对应。**你现在不翻论文，能当场告诉我——中国中车案例主要检验了哪个理论？宝钢武钢呢？美的-库卡呢？** 如果三个案例分别对应三个不同理论，那第四个理论是为谁准备的？如果对应不上，这四个理论在论文的论证链条中是必要的骨骼，还是可以删掉的装饰？

**与论文设计的关系**：这是我作为设计者在 Phase 1 埋下的结构性期待——理论不是引言里的摆设，而应在案例实证中逐一对位。Phase 7 回溯时已经发现"理论回扣悬置"是执行偏差的典型表现，但终稿未修复。

**考察意图**：测试学生对"理论-案例"映射关系的清晰认知，以及其是否意识到理论框架的冗余会稀释论文的论证力度。

**预期回答方向**：好的回答应至少能给出"中车-北车对应效率理论（规模效应/消除同业竞争）+ 市场势力理论（全球份额提升）；宝钢-武钢对应效率理论（去产能/采购协同）+ 交易费用理论（内部化）；美的-库卡对应代理理论（高溢价/商誉减值风险）+ 效率理论（技术获取）"的清晰映射；能承认某个理论在现有三案例中覆盖薄弱；若主张删掉某一理论，需给出替代方案或论证其确实无助于解释任何案例。

### Q3 [学术诚信、数据演进与透明性]

**问题**：我在 Phase 7 回溯时认领了一个设计失误：Phase 1 完全没有定义"数据溯源规则"——没有规定什么样的算真实数据、什么样的算估算、以及标注方式。这个缺位是后来学术诚信红线事件的根因：Phase 4 将模拟数据标注为 Wind/CSMAR 来源，被 5-A 判为学术不端嫌疑。**你现在作为作者回答我：如果我在 Phase 1 就强制要求"所有非直接来自数据库或年报的数据必须标为模拟，并附计算底稿链接"，你的这篇论文在质量上会变得更好，还是更受限？** 不要只答"会更好因为更诚信"——我想知道，这个约束会不会逼你放弃某些现在写了但写不干净的分析？

**与论文设计的关系**：这不是通用评审能问出的问题——它需要同时知道 Phase 1 设计缺位、Phase 5-A 红线事件、Phase 6 修改逻辑和 Phase 7 根因上移的全链条信息。设计者当评委的优势就在这里。

**考察意图**：测试学生对"学术诚信不是事后贴标签，而是研究设计的内生约束"的理解深度，以及其对论文中哪些部分真正"站得住"、哪些部分"在沙地上盖楼"的自我认知。

**预期回答方向**：好的回答应坦诚承认 CAR 事件研究和部分杜邦分解因子在严格溯源规则下可能无法展开（因为缺乏日度行情和细分科目数据）；能具体指出"表 4 中 ROE 拆解到净利率×周转率×权益乘数时，某些子指标因缺少合并前南北车单独年报的细分数据，只能是基于行业均值的推断"；若回答"毫无影响，全部可复现"或"完全写不了"，均属极端，不合格。

### Q4 [研究设计的完整性与案例选择逻辑]

**问题**：回到 Phase 1 的设计原点，我在方案中没有明确定义"成功"的标准——我只写了"成功案例研究"五个字，然后推荐了一堆政策标杆性的重组事件。你在终稿里实际上是怎么操作化"成功"的？是 CAR>0？是 ROE 没崩？是交易获批了、股票复牌了、没有退市？**如果华泽钴镍（2014 年借壳上市，后来退市）也算"完成了重组"，你会不会把它放进这篇论文作为成功案例？** 你的论文 6.3 节承认了"幸存者偏差"，但承认偏差不等于你已经解决了偏差——你选一个失败案例进来，你的核心结论还成立吗？

**与论文设计的关系**：Phase 1 的设计缺口直接导致了终稿对"成功"标准的模糊处理。主席知道这不是学生的写作失误，而是设计蓝图本身在起点就留下的空白。

**考察意图**：测试学生对其研究对象的操作化定义是否自觉，以及其是否理解"案例研究的外部效度取决于选择标准是否透明、是否可反驳"。

**预期回答方向**：好的回答应明确指出终稿对"成功"的隐性定义是多维度的（财务绩效改善+战略协同落地+市场反应正面+整合持续推进），而非单一指标；能承认华泽钴镍若按"交易完成"标准会被纳入、但按"长期绩效可持续"标准会被排除；能进一步说明加入失败案例会迫使结论从"成功的关键是 A+B+C"弱化为"A+B+C 提高成功概率，但非充分条件"。仅回答"我没考虑失败案例所以没放"属于不合格。

### Q5 [学术诚信、数据演进与透明性]

**问题**：这篇论文在数据状态上经历了三次版本跃迁：v1 是全模拟数据但标注为真实来源；v2 是全模拟数据但如实标注为模拟；v2.2（akshare 升级后）是混合数据，核心营收/净利润标了 [R]、但 CAR 和杜邦分解中的部分子指标仍标 [S]。**如果一个外部审稿人追踪了这个演进轨迹，他对这篇论文的信任是在增强还是在削弱？** 你现在的数据溯源链条——从 akshare API 调用 → CSV 文件 → Python 计算 → 表 4 单元格——能不能在答辩现场复现？如果我现在要求你打开原始 CSV，指给我看表 4 里中国中车 2015 年营业收入 2419.13 亿元这个 [R] 数字在 akshare 返回的哪一行哪一列，你做得到吗？

**与论文设计的关系**：主席亲历了 v1→v2→v2.2 的全部修改决策（M01、R1、数据溯源升级），知道每一个 [R]/[S] 标签背后都不是"作者主动选择"，而是流水线纠错的结果。

**考察意图**：测试学生对数据可复现性的实操认知，以及其是否理解"信任不是声明出来的，是流程保证出来的"。

**预期回答方向**：好的回答应坦诚说明 akshare 数据已导出为 CSV 并保留在本地，能解释从 API 返回字段到表 4 单元格的映射逻辑（如akshare的 `stock_financial_report_sina` 接口返回的营业收入字段对应哪一列）；能承认 CAR 事件研究因依赖模拟行情数据而不可复现；对"信任增减"问题，好的判断是"从 v1 到 v2 信任急剧上升（因为诚实），从 v2 到 v2.2 信任温和上升（因为有真实数据锚点），但 CAR 仍悬置意味着信任未达封顶"。回答"全部可复现"或"我没保存 CSV"均属不合格。

### Q6 [结构性自我认知]

**问题**：这是我最看重的一个问题。给你一次重来的机会——从零开始写这篇论文。**你可以改变任何一个设计层面的决策：案例选择、章节结构、方法选择、数据策略、理论框架，但只准改一个。** 你改什么？为什么选这个而不是别的？比如，你是要换掉那个让你结构失衡的第三案例，还是要在 Phase 1 就强制嵌入数据溯源规则以避免后来的诚信危机，还是要砍掉 3.2 的中美制度对比腾出篇幅做真正的跨案例比较，还是别的什么？你的选择必须告诉我一件事：你到底是这篇论文的"执行者"，还是"思考者"。

**与论文设计的关系**：Phase 7 终评给了 82% 执行度，剩下 18% 的未实现中有合理取舍也有设计失误。主席比任何人都清楚这 18% 里每一项的根因。这个问题是 Phase 7 回溯逻辑在答辩场景中的终极落点。

**考察意图**：测试学生能否在多个合理选项中进行优先级排序，并给出系统性的改进逻辑而非泛泛而谈。这是从"执行者"到"思考者"转变的核心标志。

**预期回答方向**：好的回答应给出单一、明确的选择（如"我会在 Phase 1 就锁定四案例而非三案例，把顺丰或一个失败案例作为第四象限"，或"我会在设计阶段就规定数据溯源规则为不可压缩的前置条件"）；能解释为什么这个选择优于其他选项（如"案例结构问题是根，其他问题是果"或"诚信规则是底线，底线不牢则全文皆危"）；能进一步推演改了这一个决策后，论文的其他部分会如何连锁改善。回答"每个都想改一点"或"我写得已经很好了没什么想改的"均属不合格。
