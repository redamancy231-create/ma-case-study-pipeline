## 三步工作流

### 第 1 步：设计项逐条追溯（核心）

把 Phase 1 设计中**每一个可核验的主张**对照 v2 实际产出，逐条判定：✅达成 / ⚠偏离（合理）/ ❌未达成 / ➕超出设计的合理新增。填入统一表格：

```
| 设计项(Phase 1) | Phase 1 原始意图 | v2 实际产出 | 判定 | 偏离是否合理 | 说明 |
|---|---|---|---|---|---|
| 标题 | ①或③择一 | 标题① + 副标题 | ✅ | — | … |
| 摘要框架(目的/方法/发现/结论, 250–300字) | … | 约?字, 含边际贡献+模拟数据披露 | | | |
| 六步分析框架(概念→制度→方法→案例→财务→结论) | … | 实际链路 | | | |
| 章节结构与字数(预估8100–8500字) | 6章 | 实际约? | | | 大幅超额是否仍服务设计目标 |
| 案例选择(案例一/二/三 + 备选顺丰) | 含美的-小天鹅 | 美的案例改为库卡 | | | Phase 2 修正,设计者复盘见第2步 |
| 文献候选(推荐10篇,含葛家澍/陈信元) | … | 实际16篇 | | | 推荐清单纳入/落空情况 |
| 图表方案(计划2张图) | … | 实际7表+2图 | | | |
| 过渡句设计(5处) | … | 实际落实处数 | | | |
| 数据分析维度(盈利/偿债/营运/成长/市场+CAR) | … | … | | | |
| 论述逻辑链(起点→制度→方法→案例→检验→启示) | … | … | | | |
```

> 目标是回答："**蓝图上画的，盖出来了吗？哪些走样、走样得有没有道理？**"

### 第 2 步：设计决策的"事后验证"（设计者的自我复盘）

作为原设计者，诚实评估当初的关键判断哪些被验证、哪些是失误、哪些本应不同：

- **被验证为正确的设计**：如六步框架被完整沿用、"三吸收合并/含一跨境非同控"对比骨架成立、五维度财务指标体系被采纳等。
- **被证伪、需认领的设计失误**：如 Phase 1 把"美的-小天鹅"判为**非同一控制**（被 Phase 2 纠正并替换为库卡）——作为设计者复盘该错误的根因与教训。
- **本应在设计阶段就更稳健之处**：如文献清单是否一开始就该要求"近5年占比"（对应 R2）；**数据来源是否本应在设计阶段就明确"真实 vs 模拟"的标注规则**——这正是后来 Phase 5-A 判定红线（模拟数据标注为真实来源）的**设计源头**。请评价 Phase 1 在此处的责任与改进方向。

### 第 3 步：设计者终评与签字

- 给出**方案执行度**结论（百分比 + 定性判断，可校准/细化 5-B 的"约 80%"，并说明依据）。
- 明确回答：从设计意图看，**v2 是否是 Phase 1 蓝图的忠实实现？是否值得作为该设计的最终交付封存？**
- **不重打质量分**——你的结论是对 84.0 的"设计维度补充注脚"，而非竞争性评分。
- 对归档的建议：维持 CLOSED / 建议补做某项（如有，须落在设计维度，且不与 R1/R2 重复）。

签字段示例：
