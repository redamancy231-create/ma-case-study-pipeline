## 3. 方法论核心

### 3.1 五条铁律

1. **每阶段配 prompt.md + config.json 双件** — 人机皆可读，防提示词漂移
2. **没有任何模型审查/评分自己做过的环节** — 自审有系统性盲点
3. **出题人与评分人分离，评分人"零卷入"** — 结构性独立
4. **审核早于撰写，总装最后做** — 领域硬伤在动笔前拦截
5. **诚信红线 + 内部一致性不可谈判** — 模拟数据绝不标真实来源

### 3.2 角色槽位模型

"角色是槽位，模型是填进槽位的人。" Playbook 定义槽位与槽位间的隔离规则；每个新项目重新分配人选。不绑定模型品牌。

### 3.3 铁律合规分析

**唯一违规点**: Phase 6（Claude Opus 4.7 修订自己的 Phase 4 产出）。config 以"陌生整合官"姿态缓解，但未改变"同一模型修订自己产出"的事实。

**可辩护的重叠**: Phase 7（Kimi 设计者做需求追溯，不是质量审查）、Phase 8B（GLM 同领域但任务性质不同——审蓝图 vs 追问成品）。

**完全合规**: Phase 5A/5B（GPT 审合规事实而非自己写的语言层；GLM 审写作规范而非自己审过的条款）、Phase 8E（Qwen 零卷入）。

### 6.2 对文档型项目的适用性评估

**优势**: 对代码部分提供完整函数调用图；文件枚举精确验证分布。

**关键局限**:
1. **核心方法论内容无法被图分析捕获** — 五条铁律/角色槽位/开卷盲答对照在图模型中完全不可见
2. **社区检测全部失效** — meta.json 登记 6 个集群，但 Cypher 查询仅返回 4 个且全标记为"Scripts"（cohesion 0.33–0.62），未识别实际功能分区（流水线设计/Phase快照/复用资产/翻译资产）
3. **Processes 仅捕获 Python 函数调用链** — Phase 1-8 的文档间"概念依赖"（如 P3 依赖 P2 的会计修正）不可见
4. **仅能索引 ~2% 的语义内容** — 2 个 .py 中的 31 个函数 vs 51 个 .md 中的全部方法论

**结论**: GitNexus 对文档型方法论文档项目的覆盖率非常有限。静态源码分析不足以评估以方法论/文档为核心的项目——需补充直接文件阅读和概念级分析。

### 6.3 IMPORTS 关系网络

19 条跨文件 IMPORTS 边中，`reference_files.md` 是枢纽节点，导入 17 个其他项目文件。无跨文件 CALLS——所有 81 条 CALLS 均在 `generate_docx_v2.py` 内部。

## 7. 关键局限与风险

### 7.1 Phase 0/9 从未实跑（元诊断不落地）

治低起点病的药方（Phase 0 规则前置 + Phase 9 闭环修复）已被写入 playbook 和模板件，但从未在真任务上执行。**第一个真正执行 Phase 0/9 的项目将是这两阶段的"测试用例"**——存在未验证风险。

### 7.2 类型迁移缺口

流水线在案例研究（B 类）上校准和验证，但 playbook 的目标用户主要是实证量化（A 类）。本项目最严重的 bug（CAR 不自洽/杜邦口径/无对照组）**全部聚在 A 类承重环节**——试点在容易的类型上验证了流水线，把真正要去的难类型留作未测区。

### 7.3 多样性增量未被隔离

独立评审的有效性（106%→68）已被强证据支持，但"换模型牌子"的增量从未被对照实验隔离。**同模型的全新独立实例 vs 不同模型**——哪个贡献了更多审查发现？这个问题没有答案。

### 7.4 阈值校准锚定在容错任务上

"84 = 修改后通过""开卷红利仅 −2.6 所以深度可信"——这些判断锚定在"本科会计案例研究"的容错空间上。搬到高标准实证论文时，阈值需要重新校准。

## 9. 结论

### 9.1 最有价值的贡献

1. **五条铁律 + 角色槽位模型** — 可移植、任务自适应的学术协作框架
2. **开卷/盲答对照实验协议** — 量化"LLM 评价的答案泄漏污染量"的原创方法
3. **事前否决位（Phase 0 kill-test）** —"最贵的错误不在执行里，在没人问'该不该做'"
4. **起点评估分析** — 4 模型多视角 + 红队 + 镜像参照，是多模型协作深度的最佳实证

### 9.2 最关键的局限

1. **Phase 0/9 从未实跑** — 修复低起点的钥匙自己没上过锁
2. **类型迁移缺口** — 在 B 类（案例研究）验证，用户目标是 A 类（实证量化）
3. **多样性增量未隔离** — "多模型 > 单模型多实例"的证据不完整
4. **外部效度 n=1** — 仅一个案例研究验证整个流水线框架

### 9.3 一句话总结

> **ma-case-study-pipeline 不是一篇成功的论文，而是一套被充分文档化、被反复自我批判、但核心主张尚未被独立验证的学术生产方法论。** 它最诚实的地方在于：它把"自己哪里不够好"写到了比"自己哪里好"长三倍的篇幅——这种自我批判的彻底性本身，就是它所主张的方法论价值的最好证明。
