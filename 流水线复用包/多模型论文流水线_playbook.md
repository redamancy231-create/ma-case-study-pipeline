# 多模型研究/研发流水线 · 复用 Playbook（v1.2 ｜ 论文 + 量化策略 两剖面）

> 从《中国上市公司并购重组成功案例研究》8 阶段实践 + 开卷/盲答对照实验中提炼、并经姊妹项目《形态匹配ETF策略》18 版交叉验证的可移植方法。
> 目标：把"多个独立模型交叉协作**生产 + 独立评审**"这套方法，搬到你下一个真实项目上——无论是**一篇论文**，还是**一条量化策略**。
> 本版**通用、任务自适配、不绑定具体选题和模型**：**共享一个方法内核**（五条铁律 + Phase 0 选题/否决 + 溯源门禁 + 评审轮换 + 诊断→修复闭环），下挂**两个执行剖面**——论文剖面（§3 阶段总览）与量化策略 R&D 剖面（§11）。
> 注：三件文件名仍叫 `多模型论文流水线_*` 是历史原因（保持引用稳定）；内容自 v1.2 起已泛化为研究/研发通用。

## 0. 怎么用这份包

本复用包三件：
- **`多模型论文流水线_playbook.md`**（本文）— 方法手册：原理、铁律、阶段总览、按类型特化的承重菜单。
- **`多模型论文流水线_playbook.json`**— 同内容的机读版，便于下个项目程序化读取阶段/门禁。
- **`阶段模板件.md`**— 每个阶段一套参数化 prompt+config 骨架，`{{占位符}}`填空即用。

**上手三步**：①读本文第 2、4 节（铁律 + 角色分配原则）；②按第 5 节确定你论文类型的承重阶段、裁剪掉用不上的；③去 `阶段模板件.md` 复制对应骨架、把 `{{}}` 填成你的选题，按命名规范存成 `phase{N}_{model}_{prompt.md,config.json}`。

## 1. 核心信念（为什么值得这么折腾）

一句话：**一个模型对自己的工作做自评有系统性盲点；多个独立模型反向交叉审查，能抓出自评看不见的硬伤。**

本项目的实证：总装者一度自评执行度 106%，而独立盲审判"退回重写(68)"，抓出了学术诚信红线（模拟数据标成真实来源）这种自评完全没暴露的问题。开卷/盲答对照进一步量化出：多模型答辩产出的"深度"大体是真的（认知维度盲答下零衰减），不是模型互相吹捧的表演。

**这套方法贵在"独立性"，不在"模型更聪明"。** 所有设计都服务于一件事：制造真正独立的视角，并防止任何一个视角既当运动员又当裁判。

## 2. 五条铁律（跨论文类型都不变）

1. **每阶段配 `prompt.md` + `config.json` 双件。** 人机皆可读，防提示词漂移、可追溯、可复跑。
2. **没有任何模型审查/评分自己做过的环节。** 自审有盲点，且会为旧决策辩护。撰稿人不审自己的稿、出题人不评自己的题。
3. **出题人与评分人分离；评分人选"零卷入"模型。** 出题人批自己出的题永远有偏差；评分官要结构性独立（没参与过任何前序环节）。
4. **审核早于撰写，总装最后做。** 领域硬伤要在动笔前拦截；总装者必须拥有全局上下文才能统一处理。
5. **诚信红线 + 内部一致性是不可谈判的。** 模拟/估算数据绝不标成真实来源；每个数字登记来源等级；报告自身也要过一致性自检（连成绩单都犯过"判语分数≠计算分数"的 bug）。

## 3. 阶段总览（Phase 0–9）｜ 论文剖面

> ⚑ 标记的 **Phase 0** 和 **Phase 9** 是本项目复盘后**新增**的两段——上一轮的教训是“病根在设计阶段没立规则”和“诊断出一堆 bug 却没改回论文”。这一版把它们补上。**v1.1 又给 Phase 0 补了第二半「事前否决位」**：来自姊妹项目（形态匹配ETF，一个量化策略 R&D）的教训——一套擅长抓执行错误的评审机制，救不了一个本该第一天就被否决的前提（详见 §6.5）。

| Phase | 角色原型 | 它干什么 | 关键门禁/产出 |
|------|---------|---------|-------------|
| **0 ⚑** | 选题与规则锁定 **+ 事前否决** | ①定选题、研究问题、成功/操作化定义；②**前置锁定数据溯源规则 + 文献时效规则**；③**由独立角色做「理论 kill-test」**——核心假设写成可证伪命题、给出最强反方论证、跑「最便宜的证伪实验」、预登记「死亡判据」 | 选题书 + 溯源四级规则 + 文献硬约束 + **否决官报告（最便宜证伪实验结果 + 死亡判据 + GO/STOP）**，均写成可校验条款 |
| **1** | 方案设计师 | 出大纲、候选案例/数据、理论框架、文献清单 | 设计蓝图；理论须能逐案例/逐假设对位，不是摆设 |
| **2** | 领域专家审核 | 在撰写前核领域硬伤（准则/法条/识别策略/数据口径） | 领域审查报告；价值最高的一环，抓 Phase 1 的硬伤 |
| **3** | 撰稿人 | 写正文主力，**留占位符**给下游 | 初稿；用 `【数据待填】/【图表N】/【编者注】` 不硬塞数据 |
| **4** | 总装/交付 | 数据补全、排版、终审、占位符统一处理 | 总装稿；总装者拥有全部上下文 |
| **5A/5B** | 交叉盲审 ×2 | 两个独立模型**反向**审自己没做过的环节 | 两份独立盲审（互不沟通）；按研究生标准 |
| **6** | 整合裁决与修订 | 收双盲审→意见映射→冲突裁决→统一修改→修订稿→终审 | 修订版 + 裁决记录；终审三档 |
| **7 ⚑** | 设计回溯 | 以原设计者身份评执行度，把残余问题的病根**上移到设计阶段** | 复盘报告；最有价值的产出是"该在 Phase 0/1 立什么规则" |
| **8** | 答辩模拟 | 出题(×3分层)→学生作答→零卷入评分 | 题单 + 答辩记录 + 成绩单 |
| **9 ⚑** | 答辩后定稿 | 把 5/6/8 诊断出的具体缺陷**真正改回论文**，或封存时列明"已知缺陷+不修原因" | 终稿 vX，或诚实的 closure note |

**裁剪原则**：这是全套骨架，不是每篇论文都要全跑。文献综述类可弱化 Phase 8 的方法论出题；时间紧时 Phase 7 可并入 Phase 6。但 **Phase 0、2、5、9 不建议砍**——它们分别守着“规则前置 + 事前否决”“撰前拦硬伤”“独立盲审”“闭环修复”，是这套方法的命门。

## 4. 角色分配原则（任务自适配，**不绑定模型**）

上一个项目用 Kimi/GLM/GPT/Claude/Qwen 各担一角，但那是**针对那个任务**的最优解，不是固定公式。换论文、换手头模型时，按下面三条重配：

1. **把每个模型放到它已被你验证最强的环节。** 例：擅长长链推理/整合的做总装+裁决；领域知识最扎实的做领域审；文笔与铺陈最好的做撰稿；最了解全局结构的做答辩主席。
2. **强制轮换以满足铁律 2/3。** 谁撰了稿，就不能审这稿；谁出了题，就不能评这题。让模型去审"自己没做过的环节"（上轮让撰稿模型去审领域、让审领域的模型去审写作，正是为此）。
3. **评分官留给"全程零卷入"的那个模型。** 它没出过任何题、没写过任何稿，才有外审专家式的结构性独立。

> 记法：角色是"槽位"，模型是"填进槽位的人"。Playbook 定义槽位与槽位间的隔离规则；每个新项目重新分配人选。

## 5. 按论文类型特化"承重阶段"（你选了通用版，这里是三套菜单）

选题定了之后，按类型把对应门禁挂到 Phase 0/2/8：

### A. 实证/量化研究（最贴你的量化背景）
承重 = **数据与识别**。重点门禁：
- 数据溯源四级 + 机器校验（见第 6 节），**接真实数据前不得下实证显著性结论**。
- 识别策略：有没有反事实/对照组？单样本前后对比 ≠ 因果，措辞用"伴随"非"带来"，必要时上 DID/合成控制并交代匹配难点。
- 事件研究陷阱（若用）：t 值构造（CAR/(σ_AR·√L)，单一估计窗一个 σ）；A股停牌→事件窗日历错配、复牌连续涨停→AR 正自相关致 t 虚高；单因子 vs FF3/Carhart 在风格行情下会把因子收益误记为效应；测收购方还是标的方要分清、基准指数要匹配被测市场。
- 指标体系含**现金流维度**（利润可粉饰、经营现金流难造假）；跨主体比较先做行业基准化，别拿一把尺子量不同行业。
- 稳健性的实质是**改方法**（市场模型→多因子、CAR→BHAR、参数→非参/bootstrap），**不是改窗宽**；"窗口越长效应越大"是泄漏警讯不是稳健证据。
- 杜邦等会计恒等式只能**描述**不能**证因果**；口径（归母 vs 全部权益等）全文统一，否则因果归因失分。

### B. 案例研究
承重 = **选择逻辑与比较**。重点门禁：
- 案例结构对称性：成功案例要不要失败/中止案例做对照？"成功"的操作化定义在 Phase 0 写死，别事后合理化。
- 区分会计/口径效应 vs 真实经济效应（如并表增长 vs 协同增长）。
- 跨案例结论强度：典型成功 ≠ 可推广；措辞从"证明成功"降到"成功的必要条件"。

### C. 文献综述/理论
承重 = **文献与论证结构**。重点门禁：
- 文献时效门禁（如近5年 ≥30%、核心期刊占比），Phase 0 立规则、Phase 5 校验。
- 检索策略可复现（库、关键词、纳入/排除标准、检索日期）。
- 论证是"梳理"还是"建构"——理论框架要能产出可检验命题，不是文献堆叠。

## 6. 数据溯源门禁（贯穿全程，**Phase 0 前置**）

上一项目最大的伤口、被三个独立信源（盲审/复盘/盲答对照）反复指向的同一处。规则：

- **四级来源标识**：`[R]真实`（数据库/年报可复核）/ `[S]模拟`/ `[E]估算`/ `[P]公开事实`。每个数字登记等级。
- **发布门禁**：`[S]/[E]` 不得支撑实证显著性结论；只能做"方法演示"。摘要/正文/图表口径必须一致（防"摘要289.34 vs 表格289.04"这类内部矛盾）。
- **可复现链**：原始 CSV（不可改）→ 清洗脚本（可复跑）→ 表格单元格，每个 `[R]` 能反查到源文件某行某列。
- **机器校验**：写一个能跑的校验器，扫到"`[S]`标成`[R]`"或"摘要/正文数字打架"就报错，挡在发布前。
- **前置**：以上不是 Phase 6 的事后补丁，是 **Phase 0 的设计件**。这是 Phase 7 复盘最硬的结论——诚信红线和文献时效的病根都在“设计阶段没立规则”。

## 6.5 事前否决位：选题/假设的理论 kill-test（Phase 0 前置）

> v1.1 新增的 Phase 0 第二半，来自**姊妹项目「形态匹配ETF策略」**的教训。Phase 0 原本只管“立规则”（溯源/时效），但两个项目合起来暴露了一个更靠前的漏洞：**最贵的错误不在任何一步执行里，而在“根本该不该做”这个没人在事前问的问题上。**

**为什么需要它（两项目的同一根病）**
- 并购重组：以“研究论文”立项，但起手框架根本不是研究框架（是教科书综述）。没人在动笔前问“这到底是不是一个研究问题”。
- 形态匹配ETF：核心假设“形态匹配能预测收益”在理论上注定失败（DTW 的时间弯折容忍恰好抹掉了预测最需要的方向性信息——一个白板 30 分钟能讲完的反方论证），实证上从 V2 就负 ICIR，却**烧了 18 个版本**才在终期点破，“放弃判据”也拖到第 14 个文件夹才写下。
- 共性：流水线在**执行质检**（抓 bug、抓过度外推、闭环修复）上已被反复证明有效，但它**结构性地看不见“premise 该不该做”**——所有评审都在问“这一步做对了吗”，没人被指派去问“这件事根本该不该做”。

**它是什么：一个独立的「否决官」角色，在烧工程/动笔前先试着把选题杀死。** 四个动作：
1. **把核心假设写成可证伪命题**：不是“研究 X”，而是“X 满足可被数据推翻的具体断言 Y”。
2. **给出最强反方论证**：站在“这注定不成”的立场，写出最锋利的理论 kill（ETF 那条 DTW 论证就是范例）。能 30 分钟讲完的“此路不通”，绝不留到第 18 版。
3. **设计并跑「最便宜的证伪实验」**：用最小成本（一段脚本、一张图、一个边际相关性）先打它一枪，**先于**全面建设。
4. **预登记「死亡判据」**：把“出现什么结果就放弃”写成可判定条款，**在拿到结果之前**写死——否则事后一定会被合理化成“再迭代一版试试”。

**门禁（不可谈判）**
- 否决官**必须是独立角色**——优选全程零卷入的那个模型（铁律 3 的同一个人），或专设红队；**绝不能是选题设计者本人**（他对这个想法有承诺，杀不下手）。
- **最便宜的证伪实验若已失败 → 停，不进入 Phase 1，不开工。** 把“诚实的负结果”在第 0 阶段就拿到，远比烧十几版后再拿到便宜。
- 若通过 → 继续，但**死亡判据全程挂着**，每轮回测/每稿对照它一次；触线即停，不许移动球门。

**和其它阶段的关系**
- 它把 Phase 7「病根上移设计阶段」的**事后**动作，变成 Phase 0 的**事前**动作——Phase 7 问“早该立什么规则”，Phase 0 否决位问“早该问这能不能成”。
- 它和“改一个测一个”互补：后者优化**局部执行**，但对一个注定的前提无能为力；否决位负责**全局前提**。两者缺一不可。

## 7. 答辩模拟 + 开卷/盲答对照协议

**答辩出题分三层、三个模型**：主席（设计层，应是最懂论文骨架的那个模型，不是最聪明的）、领域专家（准则/方法层）、方法论专家（实证/统计层）。三人独立出题、领域不重叠。学生（撰稿模型，对论文有作者级熟悉度但没参与审稿）逐题答。评分官（零卷入模型）双层评分（逐评审维度60% + 跨维度综合40%）+ 自检锚定。

**开卷/盲答对照（评价器的自校准，强烈建议保留）**：
- 学生作答时**必须对考官的「考察意图/预期回答方向」盲**——只给纯题干。否则是开卷应考，分数虚高、测的是"复述评分细则"而非真实理解。
- 跑两遍：开卷（含答案键）+ 盲答（剥答案键），**分差 = 答案泄漏污染量**。本项目量出仅 ≈3 分且全在"操作性回忆"层、认知维度零衰减——这才证明了多模型答辩的深度大体真实。
- 协议要点：评分官两阶段——Stage 1 不看对照基线先冷评锁分，Stage 2 才算分差（防锚定）。

## 8. 内部一致性 & 诚信红线（self-check，别只盯论文）

- **报告自身也要过一致性自检**：本项目两份成绩单都犯过"判语写的分 ≠ 最终表算的分""加权算式记法错"的 bug。凡是有数字结论的产出，发布前自己核一遍勾稽。
- **诚信红线**：模拟数据绝不冒充真实来源（这是上一项目被判"学术不端嫌疑"的根因）。宁可论文更短、结论更弱，也要每个数字诚实标级。
- **递归怀疑**：连评分官自己都有盲点（它给方法论判"零红利"，但记录显示盲答确实丢了精度）。所以"谁来监督监督者"没有终点——保留一层人工/独立复核。

## 9. 诊断→修复必须闭环（上一项目栽在这）

这套流水线极擅长**诊断**，但上一项目卡在了"无限诊断、零修复"：Phase 5/7/8 反复抓出商誉勾稽、杜邦口径、CAR 不自洽，可论文 v2 到最后还带着这些 bug。

**铁规**：每一轮诊断都要么进 Phase 9 真正改回论文（产出终稿 vX），要么在封存时把"已知缺陷 + 不修原因"写成 closure note。**诊断不落地 = 白诊断。**

## 10. 快速上手（把它套到你的毕业论文）

1. 建新项目目录，把本复用包整个 `流水线复用包/` 拷进去。
2. 跑 **Phase 0**：用 `阶段模板件.md` 的 Phase 0 骨架，填进你的选题、研究问题、成功/操作化定义，**当场把溯源规则和文献时效规则写成可校验条款**。
3. 按第 5 节选你论文类型的承重菜单，把对应门禁挂到 Phase 0/2/8。
4. 按第 4 节把手头模型分配进各角色槽位，记好隔离关系（谁不能审/评谁）。

## 11. 第二执行剖面：量化策略 R&D（论文之外的另一条产线）

> v1.2 新增，来源：姊妹项目《形态匹配ETF策略》18 个版本的实践。**共享内核不变**（五条铁律、Phase 0 选题+否决、溯源门禁、评审轮换、诊断→修复闭环），变的只是**执行半段的阶段**和**领域门禁**——论文的"撰稿/盲审/答辩"换成策略的"编码/回测/消融/幂等"。

### 11.1 为什么是"剖面"而不是"另一个包"（决策记录）

> 这一节本身是 v1.2 的"为什么这么做"记录——免得日后有人（包括你自己）再纠结要不要 fork 一个独立的策略包。

- **采用（A）**：把策略 R&D 作为现有包的**第二执行剖面**写进来，只固化两样最值钱、最易流失的东西——**阶段骨架 + 量化专属失败模式清单**；**不**预制整套 prompt/config 模板。
- **不选（B）完整重构 + 写全策略模板**：那会造出一**批从没在真任务上跑过的模板**——正是 ETF 项目用 18 个版本警告的"验证前提之前就烧工程"。模板该在你真起一个策略项目时、对着真任务实例化，不是现在凭空写。
- **不选（C）什么都不做、等真项目再说**：策略 R&D 的**失败模式清单**是用 18 个版本踩坑换来的、**会过期**的知识——现在不写下来，下次就得重踩一遍。它便宜、召回价值高，所以"什么都不做"会白白丢掉它。
- 一句话：**固化会过期的（清单+骨架），推迟没验证的（模板）。** 这本身就是 §6.5 否决位精神用在"要不要建这个包"上的一次应用。

### 11.3 量化策略专属门禁清单（18 个版本买来的，最易流失，务必带走）

> 这是 §5 菜单 A（实证/量化）向**可执行策略 R&D（代码 + 回测）**的延伸；重叠处，§5 是论文框架、本节是代码/回测框架。

**（1）泄漏 / 未来函数（最致命、最隐蔽）**
- **标签-执行对齐**：标签用 T 日信息，执行就必须 T+1 可得（ETF 栽过"标签用 T 收盘、实盘 T+1 开盘"的系统性错配）。
- **滚动窗含当日**：`x[-252:]` 若含当日就是未来函数，应 `x[-253:-1]`。
- **in-sample 回退**：NaN/异常分支偷偷回退到全训练集 in-sample 预测；Stacking 元学习器在训练集自身 `predict_proba` 生成元特征 = 泄漏。
- Walk-Forward 用**固定窗**不用扩展窗；每折**独立** fit scaler。

**（2）幂等性 / 代码纪律**
- 信号层必须**确定性**（IC/ICIR/特征重要性复跑一致到小数点后 N 位）；执行层非确定性（撮合引擎）要**量化出噪声底**，而非假装没有。
- **改一个常量，全局搜联动点**：ETF 把 `IC_WINDOW` 60→12 却漏改就绪阈值 `>=20` → 整个 IC 体系变死代码。
- `except: pass` 是 **P0 级代码异味**，能让致命 bug 跨多版本隐身。
- **单位一致**：`VOL_ABSOLUTE_CAP`（日度 σ）拿去和年化波动率比 → 100% 或 0% 触发。

**（3）实验设计**
- **改一个测一个**：单次只动一个模块，否则无法归因（ETF V3.2 同改三模块 → 全版本最差且无法定位）。
- **消融矩阵**：全特征 vs 各子特征组，找出谁真有贡献、谁是噪音；当心把"线性加性稀释"误读成"特征拮抗"。
- 消融/审计报告**自身**也要被独立复核（ETF R5 初版填错数据、谎报代码逻辑，复核后结论反转）。

**（4）信号有效性 / 迁移检验（最高价值的一条）**
- **迁移检验**：把信号搬到**另一个 universe** 验证（ETF→个股）。ETF 上 sim_variance RankIC +0.1363，个股上 −0.0104、整体 ICIR +0.0002——**指数上的"信号"很可能是成分股聚合的统计假象**。这是抓"假 alpha"最便宜最狠的一刀，强烈建议每个信号都过一遍。
- **显著性**：点估计不可信，要 t 检验/置信区间；"无法拒绝 IC=0" ≠ "证明 IC=0"，别反向过度宣称。
- 多版本筛选放大数据挖掘偏差 → Deflated Sharpe / White Reality Check / 至少声明"这是第 N 次试验取最大值"。

**（5）稳健性 / 归因 / 成本**
- **稳健性 = 改方法**（市场模型→FF3、CAR→BHAR、参数→bootstrap），**不是改窗宽**；"窗越长效应越大"是泄漏/漂移警讯。
- **归因要拆**：把 Beta + 风控择时 从"信号 alpha"里剥出来（ETF 的 4.55% 主要是 Beta+风控，真正的形态 alpha <1pp）；别把通用风控层的功劳算到你的卖点上。
- **成本现实**：换手率 × 手续费要进账（ETF 10 倍换手、手续费吃掉收益 8.5%）；指标含**现金流维度**（利润可粉饰、现金流难造假）。
- **基线**：永远对比被动基准 + 一个朴素替代（如"货基+指数"）；低回撤可能只是个通用波动率 overlay 的功劳，不是你的策略。
