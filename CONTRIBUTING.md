# 参与贡献 · Contributing

感谢关注！这是学术流水线演示项目，贡献方式与代码项目有所不同。

## 贡献方式

### 报告问题
- **勘误/笔误** → [提交 Issue](../../issues/new?template=methodology-question.yml)（标注"勘误"标签）
- **方法论疑问** → [提交 Issue](../../issues/new?template=methodology-question.yml)

### 提交改进
1. Fork 本仓库
2. 创建分支: `git checkout -b fix/description`
3. 修改后发起 Pull Request

> **本项目为 CLOSED-FINAL 状态。** 修改论文内容、流水线阶段产物、已知缺陷的 PR 不会被合并。文档错误修正和方法论描述改进欢迎提交。
>
> **📂 计划 Fork 后做大幅修改？** 先读 [`docs/fork-modification-directions.md`](docs/fork-modification-directions.md) —— Fork 修改方向全景分析（v2.0），覆盖 12 个方向、决策树、实现门槛排序和反模式清单。

## 行为准则
- 批评对准内容，不针对人
- 欢迎反例和反面证据
- 演示项目——已知论文含 3 处标注缺陷，请先读 README

## 许可
- 文档、数据、图表 → CC BY 4.0；代码（`scripts/*.py`、`.githooks/pre-push`）→ MIT。详见 [`LICENSE`](LICENSE) 和 [`LICENSE-CODE`](LICENSE-CODE)。
- 贡献内容默认继承对应文件类型的许可。
